# Выкладка на сервер (корень сайта)

## 1. Сборка проекта

```bash
npm install
npm run build
```

В папке **`dist/`** появится готовый к выкладке сайт:
- `index.html` — точка входа
- `assets/` — JS и CSS
- `images/` — картинки из `public/images/`
- `.htaccess` — правила для Apache (SPA + кэш)

## 2. Загрузка на сервер (сервер смотрит в корень проекта)

- Загрузите **весь проект** в корень сайта на сервере (включая папку `dist/` и файл **`.htaccess`** в корне проекта).
- Документ-рут сервера = **корень проекта** (ничего менять не нужно).
- В корне проекта лежит **`.htaccess`**: он перенаправляет все запросы в папку `dist/` (главная и маршруты SPA — на `dist/index.html`, файлы `/assets/`, `/images/` — из `dist/`).
- Перед выкладкой обязательно выполните **`npm run build`**, чтобы папка `dist/` была актуальной.

## 3. .htaccess (Apache)

Файл `.htaccess` уже лежит в `public/` и при сборке копируется в `dist/`. В нём:

- **RewriteRule** — все запросы, кроме существующих файлов/папок, перенаправляются на `index.html` (работает клиентский роутинг React).
- **mod_expires** — кэш картинок и статики (если модуль включён).
- **mod_deflate** — сжатие ответов (если модуль включён).

Если на сервере нет `mod_rewrite`, включите его. Для Nginx используется другая конфигурация (отдельно).

## 4. Проверка

После выкладки откройте в браузере:
- главную: `/`;
- внутренние страницы (например, `/catalog`, `/news`) — они не должны отдавать 404, контент должен открываться без перезагрузки страницы.

## 5. Локальная проверка сборки

Перед загрузкой на сервер можно проверить сборку локально:

```bash
npm run preview
```

Сайт откроется на `http://localhost:4173` (статически, как на сервере).
